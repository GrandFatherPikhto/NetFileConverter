# NetFileConverter — Универсальный конвертер нетлистов (Protel2 / KiCad)

**NetFileConverter** – это легковесное Windows-приложение на базе .NET 8, которое работает в системном трее, отслеживает изменения в выбранных папках и автоматически преобразует файлы нетлистов в упрощённые списки соединений (`_net.txt`), перечни компонентов (`_bom.txt`) и графы для Graphviz (`_net.dot`). Поддерживаются два формата: **Protel2** (Altium) и **KiCad** (s‑expression netlist).

## 🎯 Основное назначение

- **Анализ схем и поиск ошибок** – быстрый просмотр цепей и компонентов без открытия тяжёлого САПР.
- **Общение с нейросетями** – вместо отправки огромных исходников (сотни мегабайт) вы передаёте компактные текстовые файлы `_net.txt` и `_bom.txt`, что ускоряет получение ответа и экономит токены.

---

## 🚀 Функционал

- **Фоновый мониторинг** – `FileSystemWatcher` следит за созданием, изменением и переименованием `.net`-файлов сразу в нескольких папках.
- **Гибкая настройка** – для каждой папки можно указать свой формат (`Protel2` или `KiCad`) прямо в конфигурации или через окно настроек.
- **Динамическая конфигурация** – изменения `appsettings.json` применяются на лету без перезапуска.
- **Уведомления Windows** – всплывающие подсказки информируют об успешной обработке или ошибках.
- **Автоматическое первичное сканирование** – при старте обрабатываются все уже существующие `.net`-файлы.
- **Интеграция в системный трей** – приложение не занимает место на панели задач, управляется через иконку в трее.
- **Полноценный инсталлятор** – создаётся с помощью Inno Setup; автоматически прописывается в автозапуск, добавляется в `AppPaths` (запуск через `Win+R → NetFileConverter`).

### Что генерируется для каждого `.net`-файла?

Все выходные файлы помещаются в подпапку `out` рядом с исходником:

| Файл | Описание |
|------|----------|
| `*_orig.txt` | Точная копия исходного нетлиста |
| `*_net.txt`  | Список цепей с пинами в формате `Имя_цепи: R1-2, C3-1, ...` |
| `*_bom.txt`  | Перечень компонентов: `Обозначение: Тип (Comment)` |
| `*_net.dot`  | Граф соединений «компонент – цепь» для Graphviz |

## 🧱 Архитектура

| Компонент | Назначение |
|-----------|------------|
| `Program.cs` | Точка входа. Настраивает хост (.NET Generic Host), инициализирует иконку в трее, управляет жизненным циклом. |
| `Worker.cs` | Фоновая служба (`BackgroundService`). Реализует `FileSystemWatcher`, парсинг нетлистов (Protel2 / KiCad), генерацию выходных файлов. |
| `SettingsForm.cs` | Окно настроек Windows Forms. Позволяет добавлять/удалять папки и выбирать формат для каждой. |
| `appsettings.json` | Конфигурационный файл. Хранит список папок с форматами. Файл находится в `%APPDATA%\NetFileConverter\`, что решает проблемы с правами при установке в `Program Files`. |

### Взаимодействие сервисов (упрощённо)

```
Program.cs → IHost → Worker
Worker → читает appsettings.json → создаёт FileSystemWatcher’ы
FileSystemWatcher → ProcessFile → парсер (Protel2/KiCad) → запись в out/
SettingsForm → сохраняет настройки → Worker перезагружается через ReloadToken
```

## 💻 Работа с приложением

### Запуск и управление

- После установки приложение автоматически запускается и сворачивается в системный трей.
- **Правая кнопка** по иконке → контекстное меню:
  - *Настройки* – управление отслеживаемыми папками и выбор формата.
  - *Выход* – завершение работы.
- **Двойной клик** по иконке тоже открывает настройки.

### Настройка папок

1. Откройте окно «Настройки».
2. Нажмите **«Добавить папку»** – выберите каталог и **укажите формат** (`Protel2` или `KiCad`).
3. Выберите папку в списке и нажмите **«Удалить»**, чтобы исключить её из мониторинга.
4. Нажмите **«Сохранить»** – настройки запишутся в `%APPDATA%\NetFileConverter\appsettings.json`, и служба автоматически перезапустит наблюдатели.

## 🛠 Разработка и сборка

### Подготовка окружения

Установите .NET 8 SDK, затем:

```bash
dotnet restore
```

### Тестовая сборка (Debug)

```bash
dotnet build
```

### Публикация (Release) – один .exe-файл

```bash
dotnet publish -c Release -r win-x64 --self-contained true /p:PublishSingleFile=true /p:PublishReadyToRun=true /p:IncludeNativeLibrariesForSelfExtract=true
```

Результат: `bin\Release\net8.0-windows\win-x64\publish\NetFileConverter.exe`

> **Важно:** `appsettings.json` из папки публикации не используется – программа при первом запуске создаст свой конфиг в `%APPDATA%\NetFileConverter\`. Это гарантирует, что обычный пользователь сможет изменять настройки даже при установке в `Program Files`.

### Сборка инсталлятора (Inno Setup)

1. Установите **Inno Setup** (бесплатно).
2. Поместите скрипт `installer.iss` в корень проекта (см. пример ниже).
3. Откройте `installer.iss` в Inno Setup Compiler и нажмите `Ctrl+F9` (Compile).
4. Готовый установщик `NetFileConverter_Setup.exe` появится в папке `.\installer_output\`.

#### Пример `installer.iss` (адаптируйте пути под свою структуру)

```iss
[Setup]
AppName=NetFileConverter
AppVersion=1.2
DefaultDirName={pf}\NetFileConverter
DefaultGroupName=NetFileConverter
UninstallDisplayIcon={app}\NetFileConverter.exe
OutputDir=installer_output
OutputBaseFilename=NetFileConverter_Setup
Compression=lzma2
SolidCompression=yes
PrivilegesRequired=admin

[Files]
Source: "bin\Release\net8.0-windows\win-x64\publish\NetFileConverter.exe"; DestDir: "{app}"
; НЕ копируем appsettings.json – он будет создан приложением в AppData

[Icons]
Name: "{commonstartup}\NetFileConverter"; Filename: "{app}\NetFileConverter.exe"
Name: "{commondesktop}\NetFileConverter"; Filename: "{app}\NetFileConverter.exe"

[Registry]
Root: HKLM; Subkey: "SOFTWARE\Microsoft\Windows\CurrentVersion\App Paths\NetFileConverter.exe"; ValueType: string; ValueName: ""; ValueData: "{app}\NetFileConverter.exe"

[Code]
function InitializeSetup(): Boolean;
var
  ResultCode: Integer;
begin
  Exec('taskkill', '/f /im NetFileConverter.exe', '', 0, ewWaitUntilTerminated, ResultCode);
  Result := True;
end;
```

## ⚙️ Примечания для разработчиков

- **Логирование**: при запуске через `dotnet run` или напрямую `.exe` вывод появляется в консоли. В режиме службы (инсталляция) логи можно увидеть через отладчик или перенаправить вывод.
- **Добавление нового формата**: достаточно реализовать метод `ParseXxxNetlist(string[] lines) -> ParseResult` и добавить условие в `ProcessFile`.
- **Путь к конфигурации**: `Environment.GetFolderPath(Environment.SpecialFolder.ApplicationData) + "\\NetFileConverter\\appsettings.json"`.
- **Автозапуск**: обеспечивается ярлыком в `C:\ProgramData\Microsoft\Windows\Start Menu\Programs\StartUp`. Удаляется автоматически при деинсталляции.

## ❓ Часто задаваемые вопросы

**Q: Где лежат настройки, если программа установлена в Program Files?**  
A: В `%APPDATA%\NetFileConverter\appsettings.json`. Вы можете открыть его в любом текстовом редакторе.

**Q: Какой формат KiCad поддерживается?**  
A: Экспорт схемы в формате `(kicad_netlist ...)` – любой версии от 6 до 10. Используйте `File → Export → Netlist` в KiCad и выберите формат «KiCad».

**Q: Можно ли добавить папку уже после того, как программа запущена?**  
A: Да, через окно настроек. Настройки применяются без перезапуска.

**Q: Почему не создаётся папка `out`?**  
A: Проверьте, что файл имеет расширение `.net` (регистр не важен), и что формат указан правильно (Protel2 / KiCad). Посмотрите логи в консоли, если запускаете вручную.
