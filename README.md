# NetFileConverter

**NetFileConverter** — это фоновое Windows-приложение, которое автоматически отслеживает изменения в указанных папках и конвертирует файлы нетлистов (схем соединений) из форматов **Protel2** (Altium) и **KiCad** в несколько упрощённых текстовых форматов, а также строит графы для визуализации.

Проект изначально был написан как утилита для инженеров-схемотехников, чтобы быстро анализировать схемы без открытия тяжёлых САПР. В ходе рефакторинга код был полностью переработан: внедрена чистая архитектура с разделением на слои, Dependency Injection, современное логирование (Serilog), промежуточная модель данных и гибкая система генераторов.

---

## Основные возможности

- **Фоновый мониторинг** — отслеживает изменения в папках с помощью `FileSystemWatcher` (с дедупликацией событий).
- **Поддержка форматов** — KiCad (S‑выражения) и Protel 2.0 (Altium).
- **Генерация отчётов** для каждого нетлиста:
  - `_net.txt` — упрощённый список цепей с пинами.
  - `_bom.txt` — список компонентов (номиналы, корпуса, предупреждения о пропущенных footprint).
  - `_net.dot` — граф соединений в формате Graphviz (для визуализации).
  - `_net.mermaid` — диаграмма в формате Mermaid (для вставки в Markdown).
  - `_net.png` — автоматическая рендеринг PNG из DOT (если установлен Graphviz).
- **Интеллектуальная фильтрация** — при генерации DOT-графа шины питания (GND, VCC, +3V3P и т.п.) исключаются по умолчанию, чтобы избежать зависаний на больших проектах (настраивается).
- **Промежуточный JSON** — для каждого нетлиста сохраняется файл `.json` с полной моделью, что позволяет переиспользовать данные без повторного парсинга (полезно для отладки).
- **Гибкая настройка** — через `appsettings.json` в `%APPDATA%\NetFileConverter\` или через графический интерфейс (иконка в трее).
- **Удобное логирование** — используется Serilog с выводом в файл `logs.txt` в той же папке и в консоль.

## Архитектура

Проект разделён на три слоя:

- **NetFileConverter.Core** — ядро: модели данных (`NetlistDocument`, `Component`, `Net`, `PinConnection`) и интерфейсы (`INetlistParser`, `IOutputGenerator`, `INetlistSerializer`). Не зависит от внешних библиотек.
- **NetFileConverter.Infrastructure** — реализации парсеров (KiCad, Protel2), генераторов (Netlist, BOM, DOT, Mermaid, PNG) и утилит (SExpressionParser, GraphvizRenderer). Зависит от Core.
- **NetFileConverter.App** — Windows-приложение (WinForms + System Tray), настройка DI, хост, конфигурация. Зависит от Infrastructure.

Все зависимости внедряются через `Microsoft.Extensions.DependencyInjection`. Фоновая служба `Worker` управляет мониторингом, парсингом и генерацией.

## Требования

- **.NET 10.0** (или выше) — Windows-версия.
- **Graphviz** (опционально) — для автоматической генерации PNG из DOT. Если не установлен, PNG просто не создаётся, остальные функции работают.
- **Windows 10/11** (приложение использует WinForms и системный трей).

## Установка и запуск

1. **Скачайте** последнюю версию из репозитория или соберите из исходников.
2. **Сборка**:
   ```bash
   dotnet build
   ```
   для публикации:
   ```bash
   dotnet publish -c Release -r win-x64 --self-contained
   ```
3. **Запуск**:
   - В режиме разработки: `dotnet run`
   - Готовый `.exe` можно запустить как обычное приложение (появится иконка в трее).

При первом запуске автоматически создаётся файл `%APPDATA%\NetFileConverter\appsettings.json` с настройками по умолчанию.

## Настройка

Конфигурация хранится в `%APPDATA%\NetFileConverter\appsettings.json`. Пример:

```json
{
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft.Hosting.Lifetime": "Information"
    }
  },
  "Graphviz": {
    "RenderPng": true,
    "DotPath": "C:\\Program Files\\Graphviz\\bin\\dot.exe"
  },
  "DotGenerator": {
    "ExcludePowerNets": true,
    "PowerNetPatterns": "GND,+,VCC,VDD,Net-(,VSS"
  },
  "WatcherSettings": {
    "SourceDirectories": [
      {
        "Path": "D:\\Projects\\KiCad\\MyBoard",
        "Format": "KiCad"
      },
      {
        "Path": "D:\\Projects\\Altium\\MyProject\\Out",
        "Format": "Protel2"
      }
    ]
  }
}
```

- **Graphviz** — настройка рендеринга PNG.
- **DotGenerator** — фильтрация шин питания в DOT-графе (можно отключить).
- **WatcherSettings** — список отслеживаемых папок и их формат (KiCad или Protel2).

Также можно редактировать папки через графическую форму настроек (клик правой кнопкой по иконке в трее → «Настройки»).

## Использование

1. Запустите приложение.
2. Добавьте папки с нетлистами через настройки.
3. Приложение автоматически обработает все `.net` файлы в этих папках (включая вложенные? — сейчас только верхний уровень).
4. В каждой папке создаётся подпапка `out`, в которую складываются все сгенерированные файлы.
5. При изменении или создании нового `.net` файла в отслеживаемой папке он будет обработан автоматически.

---

## Сгенерированные файлы

Для каждого `myboard.net` в папке `out` появятся:

- `myboard_net.txt` — список цепей.
- `myboard_bom.txt` — спецификация компонентов.
- `myboard_net.dot` — граф для Graphviz.
- `myboard_net.mermaid` — диаграмма Mermaid.
- `myboard_net.png` — PNG-изображение (если Graphviz установлен и включён).
- `myboard.json` — промежуточная модель данных (для отладки).

## Изменения (CHANGES)

### Рефакторинг (2026‑06)

- **Полная переработка архитектуры**:
  - Выделено ядро (`Core`) с моделями и интерфейсами.
  - Внедрён Dependency Injection (Microsoft.Extensions.DependencyInjection).
  - Фоновая служба `Worker` теперь использует интерфейсы и легко тестируется.
- **Промежуточная модель данных** — введён `NetlistDocument` и JSON-сериализатор (сохраняется рядом с отчётами).
- **Парсеры вынесены в отдельные классы** (`KiCadParser`, `Protel2Parser`) и реализуют `INetlistParser`.
- **Генераторы** (`NetlistGenerator`, `BomGenerator`, `DotGenerator`, `MermaidGenerator`) вынесены в отдельные классы, реализуют `IOutputGenerator`.
- **Graphviz интеграция**:
  - Добавлен сервис `GraphvizRenderer`, который вызывает `dot.exe` через STDIN (без проблем с правами доступа).
  - Автоматическая генерация PNG из DOT (опционально).
  - **Фильтрация шин питания** в DOT-графе — исключаются цепи, содержащие `GND`, `+`, `VCC`, `VDD`, `Net-(` и т.д. (настраивается), что решает проблему зависаний на больших проектах.
- **Логирование** заменено на Serilog (пишет в консоль и в файл `%APPDATA%\NetFileConverter\logs.txt` с ротацией по дням).
- **Удалён старый код** (NetlistProcessor, KiCadWatchdog, FileLogger, старый Worker и т.д.) — теперь проект поддерживается легко.
- **Настройки** — `appsettings.json` теперь единый источник конфигурации (хранится в `%APPDATA%`), поддерживается перезагрузка на лету.
- **Улучшена обработка ошибок** — все исключения логируются, приложение не падает (если возможно).
- **Обновлены зависимости** до .NET 10.
